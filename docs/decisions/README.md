# Architecture Decision Records (ADR)

이 디렉터리는 아키텍처 설계와 관련된 결정 사항을 기록하기 위한 공간입니다. 실행 코드나 하드웨어 프로토콜 없이, **결정의 배경과 근거**를 남기는 데 집중합니다.

## 작성 규칙
1. `NNN-title.md` 형식의 파일명을 사용해 정렬 가능하게 유지합니다.
2. 회사명, 브랜드, 실제 엔드포인트와 같은 민감 정보는 포함하지 않습니다.
3. 렌더러/프리로드/메인 계층에서의 역할 분리를 명확히 기술합니다.
4. 하드웨어 연동은 인터페이스나 주석 수준에서만 언급합니다.

## 템플릿
새로운 ADR을 작성할 때는 `000-template.md`를 복사하여 사용합니다. 상태, 문맥, 결정, 근거, 결과 섹션을 모두 채워주세요.
